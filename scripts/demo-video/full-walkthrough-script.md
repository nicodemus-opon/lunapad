Target length: 8-12 minutes. Actual generated length: about 11 minutes.

1. Orient around the main app shell: workspace, notebooks, data, metrics, dbt, AI, and share surfaces.
2. Run the Sales Analytics Demo and show the SQL-led notebook loop.
3. Live-edit `growth_analysis` to show SQL cell references and reruns.
4. Tour result views and chart variety.
5. Show the visual pipeline as an optional non-SQL authoring path.
6. Cover upload/data ingest.
7. Cover external connections without exposing credentials.
8. Cover Python cells honestly: they unlock only with a real project folder and ready Python worker.
9. Cover AI as a reviewable diff workflow.
10. Cover dbt project mode: promote, compile, run/test, lineage, and scheduling.
11. Cover collaboration: comments, mentions, review inbox, and presence.
12. Cover sharing, published sites, admin, API keys, and MCP/REST automation.
13. Close on Report view.

Key framing: full local deployment unlocks the server-backed features; demo mode intentionally locks them.
